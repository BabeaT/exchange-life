# 隐私与权限矩阵

## 1. 角色

- **用户甲**：当前内容所有者时拥有本人权限；作为对方时只拥有送达后的接收权限。
- **用户乙**：权限结构与用户甲对称。
- **系统业务服务**：执行身份、状态、送达、版本和权限校验；不以业务便利扩大内容用途。
- **AI 整理服务**：仅处理主动触发者本人的最小必要材料。
- **视角交汇服务**：只处理双方最终公开信件与主动公开附件的固定快照。
- **记忆检索服务**：未来能力；按查询者与索引范围检索，当前 Demo 仅模拟。

## 2. 权限动作定义

| 动作 | 含义 |
|---|---|
| 读 | 能取得完整内容，而非仅看粗粒度状态 |
| 编 | 能修改内容或其本人版本 |
| 发 | 能将内容提交为正式信件或回应；不代表能绕过确认 |
| AI | 能作为指定 AI 能力的输入 |
| 记忆 | 能进入共同记忆实体 |
| RAG | 未来能进入对应权限索引并被检索 |

标记：`✓` 允许；`条件` 仅满足说明条件时允许；`—` 禁止。用户甲/乙列以“用户甲是材料所有者”为例；乙拥有自己材料时完全对称。

## 4. 按动作核对

### 可读取

- 用户只读自己的私人材料，以及送达给自己的正式信件和双方共同结果。
- 系统服务只在完成明确业务操作时读取必要字段。
- AI 整理服务只读触发者本人的选定材料。
- 视角交汇服务只读双方最终公开快照。

### 可编辑

- 私人原始材料、草稿和 AI 候选稿仅所有者可编辑。
- 对方正式信件、交汇和共同记忆不能被任一方改写成新的共同事实。
- 正式寄出前修改会使旧确认失效；送达后的修改/撤回规则仍待产品确认。

### 可发送

- 只有内容所有者能确认并寄出自己的正式信件、公开附件和情感回应。
- AI、系统服务和对方均不能代替本人确认或发送。

### 可用于 AI

- 私人材料只用于本人主动触发的整理类能力。
- 共同 AI 只使用最终公开信件和主动公开附件。
- 一个能力获得的材料不得自动转授权给另一个能力。

### 可进入共同记忆

- 原始材料、私人草稿和 AI 候选稿永不直接进入。
- 只有双方正式信件、公开附件、双方可见回应、交汇结果及其元数据可进入；共同记忆碎片仍需满足完整业务条件。

### 可被 RAG 检索

- 当前 Demo 不建立真实 RAG 索引。
- 未来私有索引只对内容本人开放；共同索引只包含双方最终公开内容。
- 情感回应默认不进入语义检索，避免推断心理；是否开放需单独确认。
- 权限过滤必须发生在召回前，并在每次查询重新校验。

## 5. 技术执行要求

- 数据库使用所有者字段、空间成员关系、送达状态与 RLS；文件使用私有桶和短时签名 URL。
- 服务端接口按能力构造输入，不允许客户端提交任意对象路径或另一用户 ID 获取内容。
- 日志默认只记录任务 ID、状态、错误码和数据引用，不记录完整私人正文或媒体。
- 交汇保存公开来源快照，记忆检索结果保存来源与权限标签。
- 缓存键必须包含用户与授权范围；退出登录后清除浏览器中的敏感缓存。

## 6. 待产品确认

- 已送达信件、公开附件和共同记忆的撤回、隐藏及删除范围；
- 账号注销后共同内容的保留与署名方式；
- 私人材料、AI 中间结果、音频和检索索引的保存期限；
- 是否允许原始录音进入本人私有 RAG，以及情感回应是否永远排除在语义检索外。
