# 《我离开家的那一年》完整 Demo 演示方案

## 1. Demo 目标

用母亲丹青与女儿林夏对同一次离家的不同记忆，完整展示“交换人生”的核心价值：双方先在私人空间独立表达，只公开本人确认的图文信件；两封信完成交换和阅读后，再由 AI 生成保留差异的双视角交汇，最终形成共同记忆。

本 Demo 使用 Word 原稿中的完整双视角故事和 14 张原图。故事地点统一为家、去机场的出租车、航站楼安检口、深圳出租屋与离开后的家。

## 2. 演示前准备

1. 在项目根目录运行 `npm run dev`。
2. 浏览器打开 `http://localhost:3000/demo-control`。
3. 点击“重置全部 Demo 数据”，确认重置。
4. 确认当前身份为“丹青”。
5. 建议使用宽度不小于 1280px 的桌面浏览器，并将缩放保持在 100%。

## 3. 推荐演示节奏

完整演示约 18 分钟。重点不是逐字读完全部长文，而是让观众看到每一步的隐私边界、确认动作和图文叙事连续性。

| 阶段 | 时长 | 要证明的事 |
| --- | ---: | --- |
| 建立共同事件 | 2 分钟 | 两个人围绕同一事件交换，但各自表达保持独立 |
| 丹青完成信件 | 4 分钟 | 原始表达私密，AI 结果须本人确认，图片按文字情节插入 |
| 林夏完成信件 | 3 分钟 | 对方不能提前看到，双方都寄出后才交换 |
| 双方读信与回应 | 4 分钟 | 逐幕阅读，可以轻回应，不形成即时聊天压力 |
| 双视角交汇 | 3 分钟 | AI 只读两封正式信，区分事实、理解和未看见部分 |
| 共同记忆 | 2 分钟 | 交汇结果可保存为长期共同记忆，但不设置打卡惩罚 |

## 4. 完整演示步骤

### A. 建立共同事件

1. 在 Demo 控制台点击“模拟对方加入”。
2. 点击“加载双方原始表达”。
3. 点击右上角“进入产品”。
4. 打开《我离开家的那一年》的双人空间。

演示话术：

> 这是母亲丹青和女儿林夏共同经历的一次离家。她们选择的是“共同事件”和“双方独立表达”。两个人知道在写同一件事，但任何一方都看不到另一方的草稿、录音、图片和 AI 工作稿。

检查点：事件范围应出现晚饭、收拾行李、去机场、黄色等候线、飞机起飞和离开后的消息，不应出现火车站或列车。

### B. 丹青完成第一封信

1. 以丹青身份点击“写下我的版本”。
2. 在写作页确认完整母亲视角原稿已载入，右上角显示“只对你可见”。
3. 点击“完成表达，交给 AI 整理”。
4. 点击“开始整理”，等待候选稿出现。
5. 强调左右两栏：左边原始表达保持不变，右边候选稿可编辑。
6. 点击“这就是我想说的”。
7. 在图文信件页查看 7 幅故事图片的自动编排，并切换一次“文字为主 / 图文平衡 / 图片为主”。
8. 点击“进入完整预览”，向下滚动，抽看“盛汤”“收行李”“机场安检”“空下来的家”四幕。
9. 点击“确认最终版本”，选择“双方都寄出后交换”，再点击“确认寄出”。

演示话术：

> AI 可以整理和编排，但不能替丹青公开。只有她确认后的文字和插图会成为正式信件。丹青先寄出后，信件由系统托管；林夏看不到正文，也看不到她写了多少字、什么时候完成。

检查点：丹青的正式信标题应为《那一天，我第一次发现孩子真的会离开》，共 7 幕；图片应与盛汤、担心、角色变化、收行李、去机场、安检和空房间对应。

### C. 林夏完成第二封信

1. 在等待页点击“切换身份，完成对方表达”。
2. 确认当前身份变为林夏。
3. 重复“写作 → AI 整理 → 本人确认 → 图文编排 → 完整预览 → 确认寄出”。
4. 在林夏预览中重点查看“凉下来的晚饭”“推开一扇门”“行李里的一点家”“飞机起飞”“深夜手机”五幕。
5. 林夏寄出后，等待页应显示“两封信已经交换”。

演示话术：

> 同一顿晚饭，在林夏那里不是挽留，而是一场必须守住勇气的告别。系统没有要求两个人写出同样的答案，只要求各自确认自己的版本。

检查点：林夏的正式信标题应为《那一天，我终于决定离开》，共 7 幕。双方第二封信寄出后应同时进入已送达状态。

### D. 林夏阅读丹青的信

1. 点击“去看看收到的信”，打开丹青寄来的信。
2. 点击“打开这封信”，确认轻柔背景音乐开始播放。
3. 不进行翻页操作，观察文字逐段出现、关联图片在对应文字之间缓慢浮现，页面自动跟随最新内容。
4. 在工具栏依次检查“暂停/继续浮现”“展开全文”“自动跟随”“音乐开关”和音量滑杆。
5. 重点停留在“我把舍不得说成了问题”和“家突然变得太大了”。故事完整浮现后点击“我读完了”。
6. 选择“我看见了”，发送回应。
7. 在后续选择页点击“就停在这里”。

演示话术：

> 收信不是聊天入口。林夏可以只用一个低压力动作告诉母亲“我看见了”，也可以暂时不回信。产品不会用未读、连续提醒或聊天流制造回应压力。

检查点：读信页只显示丹青最终确认的图文信，不显示丹青的原稿、AI 对照稿或私密素材；无需手动翻页；图片保持 16:9 比例；离开读信页后背景音乐停止。

### E. 丹青阅读林夏的信

1. 回到 `http://localhost:3000/demo-control`。
2. 将当前身份切换为“丹青”，点击“进入产品”。
3. 打开林夏寄来的信，让 7 幕内容自动浮现；可以降低音量，但不要手动翻页。
4. 故事完整浮现后点击“我读完了”，选择“轻轻靠近”并发送。
5. 此时页面应提示“两个视角都已经抵达”，点击“查看两个视角”。

演示话术：

> 只有两个人都读完，双视角交汇才具备生成条件。AI 的输入被严格限制为双方最终确认并公开的两封信。

### F. 生成双视角交汇

1. 点击“生成视角交汇”。
2. 在“五节点对照”中依次浏览：晚饭、反复询问、收行李、机场安检、离开后的夜晚。
3. 每个节点检查“共同事实”“不同理解”“当时没有看见”三栏。
4. 切换到“简明时间线”，展示同一内容的快速浏览方式。
5. 浏览页面下方四组结论，然后点击“形成记忆碎片”。
6. 点击“查看记忆碎片”。

演示话术：

> 交汇不是判谁对谁错，也不把两个人改写成同一种感受。它把共同事实、各自理解和当时没看见的部分分开呈现。黄色等候线两边的沉默，在母女那里可以同时拥有不同含义。

检查点：交汇页不得声称读取了原始录音、原图、私人转写或未确认草稿。

### G. 保存共同记忆

1. 在记忆碎片页查看交叠记忆插图、共同事实和仍然保留的不同。
2. 点击“保存到记忆树”。
3. 点击“在共同记忆树上看看它”。
4. 打开“离开家的那一年”记忆果实。
5. 返回记忆树，点击一次“轻轻浇水”。

演示话术：

> 共同记忆不是把两封信压缩成一个标准答案，而是把这次相互看见保存下来。浇水只是一次温和的照料，没有连续天数、积分、排行榜，也不会因为不来而枯萎。

## 5. 8 分钟快速演示

当时间有限时，在 Demo 控制台按下面顺序操作：

1. 重置全部数据。
2. 点击“模拟双方发送”。
3. 以丹青身份进入产品，阅读林夏的 7 幕图文信并回应。
4. 回到 Demo 控制台，切换为林夏。
5. 进入产品，阅读丹青的 7 幕图文信并回应。
6. 进入视角交汇，生成结果。
7. 形成记忆碎片并保存到记忆树。

这条路径跳过创建和 AI 整理，但完整保留“读信 → 回应 → 交汇 → 记忆”的核心产品价值。

## 6. 可选支线：我先告诉对方

这条支线用来证明产品不会强迫收信人回信，也不会在只有一封信时生成双视角交汇。

1. 回到 Demo 控制台并重置数据。
2. 点击“模拟对方加入”。
3. 点击“切换交换方式”，确认显示“当前：我先告诉对方”。
4. 点击“加载双方原始表达”，保持丹青身份进入产品。
5. 让丹青完成并寄出一封信。此时信件应直接送达林夏，不等待林夏先完成表达。
6. 切换为林夏，阅读并发送一个轻回应。
7. 在回应后的选择页点击“就停在这里”。

检查点：页面允许林夏稍后再决定是否回信；只有一封正式信时，不应生成双视角交汇或共同记忆碎片。

## 7. 演示后记录问题

跑完后按下面四类记录修改意见：

- 故事准确性：文字、人物、地点、时间或双视角是否与 Word 原稿不一致。
- 图文匹配：某张图是否放错段落、裁切不当或出现重复含义。
- 操作节奏：哪一步太长、按钮难找、状态变化不清楚或需要额外解释。
- 情绪与隐私：文案是否给人催促、评判、强迫和解或暴露私人材料的感觉。

每条问题建议记录“所在页面、当前身份、执行动作、预期结果、实际结果、截图”，这样可以直接定位和修改。
