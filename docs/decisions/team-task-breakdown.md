# 《交换人生》团队任务分工与需求映射

## 1. 使用方式

本文件把比赛版与正式 MVP 的主闭环拆成 24 个一级任务。每个任务只有一名主负责人；“配合人”负责提供接口、设计或联调支持，不承担最终交付责任。

飞书多维表格主表：[交换人生｜需求分工与任务排期](https://my.feishu.cn/base/EQSTbbtFuahk7nshDPAcNvrinEc)。

工作量仅用于排期初估：`S` 为半天左右，`M` 为 1 至 2 天，`L` 为 3 至 5 天。进入开发后再把一级任务拆成代码级子任务。

飞书多维表格中每个 `Txx` 对应一条记录，字段为：任务 ID、阶段、任务名称、主负责人、配合人、优先级、工作量、前置任务、任务详细描述、需求详细、具体交付物、验收标准、PRD 对应部分、功能需求对应部分、页面与技术对应部分、状态。
