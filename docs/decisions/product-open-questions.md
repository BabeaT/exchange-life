# 《交换人生》产品未决问题

本文件用于产品讨论和决策跟踪。完整背景与功能影响见 `docs/product/complete-prd.md` 第 18 节。

状态说明：`待讨论`、`已决定`、`需验证`。每次拍板后记录决定、原因、日期和影响文档。

## P0：进入正式开发前决定

| ID | 问题 | 建议默认方案 | 状态 |
|---|---|---|---|
| D-P0-01 | 产品是否承诺“和解” | 不承诺和解，以“看见、理解、重新讲述”为主 | 待讨论 |
| D-P0-02 | 双方模式的 B 是否要明确接受 | B 必须明确接受，才成为共同创作者 | 待讨论 |
| D-P0-03 | 一段人生怎样结束 | 创建时选择结束日期或手动结束；延期需双方确认 | 待讨论 |
| D-P0-04 | 双方何时看见彼此源稿 | 双方提交后先生成并阅读共同绘本；源稿按 `hidden/after_storybook` 策略控制 | 已决定 |
| D-P0-05 | 共同绘本怎样确认 | 双方分别确认，未双确认不进入共同记忆 | 待讨论 |
| D-P0-06 | 到期一方未提交怎么办 | 不自动公开；允许延期、各自保存或已提交方转单方 | 待讨论 |
| D-P0-07 | AI 上下文保留多久 | 按项目保存，用户可清除，普通日志不存正文 | 待讨论 |
| D-P0-08 | 已送达内容是否可撤回 | 不删除对方已接收副本；允许标记作者撤回并停止后续共享 | 待讨论 |

## P1：正式 MVP 前决定

| ID | 问题 | 建议默认方案 | 状态 |
|---|---|---|---|
| D-P1-01 | 单方人生记录能否边写边分享 | 首版只在阶段结束后统一送达 | 待讨论 |
| D-P1-02 | 人生周期最长多久 | 1 至 90 天 | 需验证 |
| D-P1-03 | 文字、图片和录音上限 | 按生成与存储成本设置软限制 | 需验证 |
| D-P1-04 | 是否生成可识别真人 | 首版使用非写实角色，不生成可识别真人 | 待讨论 |
| D-P1-05 | 共同结果争议如何结束 | 保留源稿；候选可标记、隐藏和重生成，不允许单方改写对方 | 待讨论 |
| D-P1-06 | 单方项目能否升级为双人 | 可以升级为“看后回写”，但不能改称互不可见共述 | 待讨论 |
| D-P1-07 | 账号注销后共同内容怎样处理 | 保留共同副本，作者匿名化或标记已注销 | 待讨论 |
| D-P1-08 | 未成年人规则 | 上线前确定年龄门槛和监护要求 | 待讨论 |
| D-P1-09 | 怎样收费 | 比赛版不收费，后续优先为高清导出或实体印刷收费 | 需验证 |

## P2：体验增强阶段决定

| ID | 问题 | 当前备选 | 状态 |
|---|---|---|---|
| D-P2-01 | 时间轮盘使用什么形态 | 双轨时间轴、旋转时间轮、章节书架 | 需验证 |
| D-P2-02 | 记忆信使是否保留 | 保留完整角色，或降为轻量送达动效 | 需验证 |
| D-P2-03 | 支持哪些导出格式 | PDF 优先，其次长图，视频后置 | 待讨论 |
| D-P2-04 | 是否接入实体印刷 | 作为独立商业阶段处理 | 待讨论 |
| D-P2-05 | 是否加入私人记忆检索 | 仅检索本人授权内容，与共同索引隔离 | 待讨论 |
