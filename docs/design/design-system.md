# 《交换人生》Web Demo 设计系统

## 视觉方向

“雾感纸本绘本 × 柔软抽象角色 × 克制的情绪叙事”。参考图仅用于提炼低饱和、留白、纸张颗粒和柔软轮廓；Demo 使用原创 CSS 场景与抽象角色，不复用参考角色、构图或艺术家作品。

## 设计变量

| Token | 值 | 用途 |
|---|---|---|
| `--paper-bg` | `#F3F0EA` | 页面背景 |
| `--paper` | `#FAF8F3` | 编辑器、信件和重要内容表面 |
| `--mist-blue` | `#CAD5D6` | 安静、等待、用户甲视角 |
| `--dusty-pink` | `#D9BDBD` | 回应、用户乙视角 |
| `--lavender` | `#BDB4C3` | 共同空间和交汇 |
| `--sage` | `#ADB89F` | 完成、成长、记忆树 |
| `--ochre` | `#C4A277` | 光、焦点和提醒 |
| `--brown` | `#6B5548` | 主按钮和主要文字强调 |
| `--text` | `#46403C` | 正文 |
| `--muted` | `#7A716B` | 辅助说明 |

标题与信件正文使用宋体气质的衬线字体；按钮、表单和状态使用清晰无衬线字体。信件表面纹理不得降低可读性。

## 组件原则

- 主按钮使用暖棕色胶囊形，次按钮使用纸面和柔和描边。
- 面板使用 16–24px 圆角、极浅阴影；正式信件保持更接近纸张的 4px 小圆角。
- 输入焦点、错误、禁用、加载和完成状态均有明确边框或颜色，不只依靠柔和氛围表达。
- 角色使用原创 CSS 不规则轮廓和两个黑点眼睛；不使用具版权识别度的参考角色。
- 动画以平移、呼吸和淡入为主；支持 `prefers-reduced-motion`，并可在设置中选择减少动态。

## 布局

- 桌面主布局：68px 顶栏、230px 轻侧栏、内容最大宽度 1320px。
- 表达、对照和信件编辑充分利用横向空间。
- 1000px 以下收起侧栏；720px 以下主要双栏改为单栏，但不模拟手机 App 外壳。

## 品牌入口排版

- “交换人生”四字作为不可拆分标题：不插入空格或 `<br>`，使用 `white-space: nowrap`、`word-break: keep-all` 和 `overflow-wrap: normal`。
- 内容容器使用 `width: min(78vw, 920px)` 与不超过视口的最大宽度；标题字号使用 `clamp(2.4rem, 8.5vw, 8.8rem)`，在 1920、1440、1280 宽度保持大标题呼吸感，在窄屏自动缩小。
- 主文案由两个完整语义行组成；只在窄屏空间不足时从显式语义断点换行，不随机拆词。
- 品牌页自身隐藏场景装饰溢出，正文容器不超过视口，因此不得产生横向滚动条。

## 产品意象边界

信件是经本人确认的个人视角载体，不是产品目的。视觉重点应从“邮寄物件”转向两个人各自的位置、视角的抵达与转换、交汇后的共同记忆；送信动画只表达视角抵达，不采用奖励化表现。
