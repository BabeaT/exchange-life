# 页面规格摘要

完整实际路由见 `implemented-page-map.md`。

## 页面层级

1. **氛围入口**：品牌页保持单一进入操作，不放功能卡和营销数据。标题“交换人生”始终单行显示；主文案固定为“我的话语，一半毫无意义；但我说出来，是为了让另一半能够抵达你。”，桌面端按两个完整语义行排版。页面不设置底部副标题或空占位。
2. **身份与加入**：进入方式、登录注册、资料、邀请码均使用独立聚焦布局。
3. **产品工作区**：首页、交换、表达、信件、记忆使用统一 AppShell。
4. **沉浸内容**：信件送达、阅读、交汇和记忆树减少导航干扰并扩大内容区域。
5. **开发辅助**：Demo 控制台独立于 AppShell，不进入正式主导航。

## 叙事载体原则

- 页面目标围绕独立表达、视角抵达、视角转换、交汇和共同记忆，而不是完成“写信任务”。
- 信件仅承载本人确认后的个人视角；信封与送信动画只在视角抵达场景使用。
- 非必要页面避免反复使用寄信隐喻，优先使用“表达、抵达、看见、交汇、记忆”等产品语言。
- 交汇页面不承诺和解、不裁判对错，也不消除双方差异。

## 通用状态

每个关键流程至少提供正常、加载、成功、错误/恢复和空状态。托管、定时、权限不足和不存在页面使用明确文案，不泄露私人内容。

## 可访问性

- 主要按钮和链接支持键盘操作及 `:focus-visible`。
- 表单字段包含显式 label；插图、角色和信件内容提供语义或 alt。
- 音频与动画不自动开始；减少动态偏好生效。
- 桌面推荐 1440×900，最窄兼容至 360px 浏览器宽度。
