# 产品核心定义

## 1. 产品核心

《交换人生》是一个面向两位熟人的桌面 Web 双人叙事产品。它让双方分别从自己的位置重新描述一件共同经历，或分享各自人生中的片段；通过独立表达、延迟抵达和有限度的 AI 整理，让彼此暂时转换观看位置，看见同一件事情在另一个人生命中的样子，并把完成的双向交汇沉淀为共同记忆。

> **明确原则：**信件是视角传递的载体，不是产品目的；产品的核心行为是独立表达、视角抵达、视角转换与共同记忆的形成。

产品闭环是：创建双人交换 → 选择叙事内容与交换方式 → 私密独立表达 → AI 整理且本人确认 → 个人视角抵达对方 → 阅读并暂时转换观看位置 → 双向条件满足后生成视角交汇 → 在保留双方差异的前提下形成共同记忆碎片。

产品不承诺双方一定和解，不判断哪一方正确，也不通过整理消除双方原本存在的身份、感受与差异。

## 2. 双人关系边界

- 每次交换仅绑定发起人 A 与受邀人 B；邀请码由熟人通过外部渠道私下传递。
- 产品不负责匹配陌生人，也不判断或强制标注双方是亲人、伴侣或朋友。
- 双人空间以双方昵称命名。双方共同拥有已完成的交汇结果，但各自始终拥有独立的私人表达区。
- 对方只能获知“准备中、已寄出、已送达”等粗粒度状态，不能看到字数、录音时长、编辑时间或草稿细节。

## 3. 共同事件与一段人生

“叙事内容”是第一个独立选择维度：

- **共同事件**：双方围绕同一件共同经历讲述各自版本。A 创建并确认中性的事件卡，B 只读并确认自己知道所指事件。
- **一段人生**：用户在一个有开始和结束条件的周期内，多次记录文字、语音和图片。双方模式拥有彼此不可见的私人时间线，周期结束且双方提交后形成双轨时间线；不要求双方讲同一事实。

两类叙事均可搭配任一种参与方式，不能把共同事件固定为双方表达，也不能把一段人生固定为单方分享。

## 4. 双方独立表达与“我先告诉对方”

“交换方式”是第二个独立选择维度：

- **双方独立表达**：双方在互不可见的私人空间完成表达。先提交的故事源稿由系统托管，两边都提交后同时揭晓，再生成共同故事和绘本。
- **我先告诉对方**：A 确认源稿并生成个人故事与绘本后送达，B 收下后可以轻回应，也可以在准备好时回写。回写不是任务，不设置倒计时或催促。

## 5. AI 在产品中的角色

AI 是受限的整理与呈现工具，不是关系裁判或代写者。它可提供上下文总结、轻量提示、语音转写、OCR、叙事整理、个人绘本、共同故事与绘本、插图与版式建议，以及语音朗读。

所有 AI 整理或生成内容必须经过本人确认。私人草稿、原始媒体与工作摘要不能进入 AI 交汇；交汇只能使用双方最终公开信件和主动公开附件。

## 6. 产品不做什么

- 不做即时聊天、评论区或连续刷表情。
- 不做陌生人匹配、公开社交广场或关系评分。
- 不做排行榜、签到、连续天数、任务奖励或树木枯萎惩罚。
- 不做心理诊断、关系裁判、动机推断或情绪操控。
- 不以 AI 自动发布、自动回信或自动替用户确认内容。
- 不要求双方同时在线，不以同时在线作为交换完成条件。
- 不把“写一封信”作为最终目标，也不把产品描述为写信工具；信件、信封和送信动画只在承载及传递已确认个人视角时使用。
