# 用户流程

## 1. 普通用户创建交换

```mermaid
flowchart TD
  A[品牌入口] --> B[进入方式选择]
  B --> C[创建一次交换]
  C --> D{账号状态}
  D -->|已登录| E[选择叙事内容]
  D -->|未登录或未注册| F[登录/注册与个人资料]
  F --> E
  E --> G[共同事件卡或一段人生周期卡]
  G --> H[选择交换方式与送达方式]
  H --> I[创建确认]
  I --> J[生成交换 ID、邀请码和双人空间]
  J --> K[复制邀请]
  J --> L[开始私人表达]
```

## 2. 邀请码加入

```mermaid
flowchart TD
  A[输入邀请码或打开邀请链接] --> B{邀请码有效?}
  B -->|否| C[显示错误与重试]
  B -->|是| D[暂存邀请码和交换 ID]
  D --> E{账号状态}
  E -->|已登录| F[绑定为参与者 B]
  E -->|未登录或未注册| G[登录/注册与个人资料]
  G --> F
  F --> H[查看事件卡或主题卡]
  H --> I[确认识别或理解主题]
  I --> J[进入对应双人空间]
```

异常包括无效、失效、已使用、自己的邀请码、交换取消或结束；邀请码回跳不得因登录注册丢失。

## 3. 四种核心组合

### 3.1 共同事件＋双方独立表达

```mermaid
flowchart LR
  A[A 确认中性事件卡] --> B[B 加入并识别事件]
  B --> C[A/B 分别私密表达]
  C --> D[A/B 各自 AI 整理并确认]
  D --> E[先寄出者进入托管]
  E --> F{双方均寄出且送达条件满足?}
  F -->|否| E
  F -->|是| G[两封信同时送达]
```

### 3.2 共同事件＋我先告诉对方

```mermaid
flowchart LR
  A[A 确认中性事件卡] --> B[A 私密表达与确认]
  B --> C[A 寄出]
  C --> D[B 收信、阅读、发表情]
  D --> E{B 自愿回信?}
  E -->|否| F[保留单方来信]
  E -->|是| G[B 围绕同一事件回信并确认]
  G --> H[A 阅读回信]
```

### 3.3 一段人生＋双方独立表达

```mermaid
flowchart LR
  A[A 确认人生周期卡] --> B[B 接受并查看周期规则]
  B --> C[A/B 在私人时间线持续记录]
  C --> D[周期结束后各自汇总并确认源稿]
  D --> E[先提交者的源稿进入托管]
  E --> F[双方提交后锁定源稿快照]
  F --> G[AI 直接生成双轨时间线和共同绘本]
  G --> H[双方校对并分别确认]
  H --> I[阅读共同绘本]
  I --> J{源稿访问策略}
  J -->|hidden| K[不开放双方源稿]
  J -->|after_storybook| L[读完后开放源稿入口]
```

### 3.4 一段人生＋我先告诉对方

```mermaid
flowchart LR
  A[A 确认人生周期] --> B[A 在私人时间线持续记录]
  B --> C[周期结束后 AI 汇总且 A 确认源稿]
  C --> D[生成个人绘本并由 A 确认]
  D --> E[A 寄出并送达 B]
  E --> F[B 阅读并轻回应]
  F --> G{B 自愿分享自己的版本?}
  G -->|否| H[保留单方故事]
  G -->|是| I[B 建立关联表达并确认]
```

## 4. AI 整理与本人确认

```mermaid
flowchart TD
  A[私人原始文字/录音/图片] --> B[用户点击完成表达]
  B --> C[AI 生成整理稿]
  C --> D[原文、整理稿和差异对照]
  D --> E{本人确认?}
  E -->|修改或补充| F[回到编辑/局部重整]
  F --> C
  E -->|确认| G{参与方式}
  G -->|我先告诉对方| H[生成个人故事、插图与版式]
  H --> I[收信人视角预览]
  I --> J{本人最终确认?}
  J -->|否| H
  J -->|是| K[允许寄出]
  G -->|双方独立表达| L[提交并托管本人故事源稿]
  L --> M{双方源稿均提交?}
  M -->|否| L
  M -->|是| N[生成共同故事与绘本]
  N --> O[双方校对并分别确认]
```

任何未确认结果均保持私密，不能寄出或参与交汇。

## 5. 送达、阅读与表情回应

```mermaid
flowchart LR
  A[满足送达条件] --> B[信使送达信封]
  B --> C[收信人主动开启]
  C --> D[阅读并恢复阅读位置]
  D --> E[结尾选择一个专属表情]
  E --> F[确认已读并保留回应]
  F --> G{是否自愿回信?}
  G -->|否| H[流程可在此结束]
  G -->|是| I[进入关联回信表达]
```

## 6. AI 视角交汇、记忆碎片与记忆树

```mermaid
flowchart TD
  A{双方故事源稿均提交?} -->|否| X[不生成共同绘本]
  A -->|是| B[服务端锁定两份源稿与公开附件]
  B --> C[按共同事件或一段人生生成共同绘本]
  C --> D[双方校对、反馈或重生成]
  D --> E[双方分别确认]
  E --> F[阅读共同绘本]
  F --> G{源稿访问策略允许?}
  G -->|否| H[保持源稿隐藏]
  G -->|是且已读完| I[开放受控源稿入口]
  H --> J[形成共同记忆碎片]
  I --> J
  J --> K[挂入双方共同记忆树]
```
